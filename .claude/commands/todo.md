# /todo - tornado 互換タスク追加スキル

`$ARGUMENTS` の内容を、現在のプロジェクトの `plan.md` に tornado 互換の形式で追加する。

## plan.md のフォーマット

```markdown
# Plan

- [ ] タスク1の説明
- [ ] タスク2の説明
- [x] 完了済みタスク
```

## 手順

### 1. plan.md の確認

プロジェクトルート（git root）の `.claude/plan.md` を読み込む。存在しなければ `.claude/` ディレクトリごと作成し、以下のヘッダーで新規作成する:

```markdown
# Plan

```

### 2. タスクの整形

`$ARGUMENTS` を分析し、適切なタスクに変換する。

**粒度のルール:**
- 1タスク = エージェントが1イテレーションで完了できる単位にする
- 大きな機能は具体的なサブタスクに分解する（目安: 3〜7個）
- 各タスクは「何をするか」が明確で、実装者が迷わない記述にする

**具体化のルール:**
- 曖昧な指示の場合はコードベースを確認し、具体的なファイルパスや関数名を含める
- 「〜を実装」だけでなく、スコープ（対象ファイル、エンドポイント、コンポーネントなど）を明記する

**分解の例:**

入力: `ユーザー認証`

出力:
```markdown
- [ ] User モデルに password_hash フィールドを追加し、マイグレーションを作成
- [ ] POST /api/auth/signup エンドポイントを実装（パスワードハッシュ化、バリデーション）
- [ ] POST /api/auth/login エンドポイントを実装（JWT 発行）
- [ ] auth middleware を実装し、保護ルートに適用
- [ ] auth 関連のユニットテストを追加
```

### 3. plan.md への追記

- 既存の `- [x]`（完了済み）の後、`- [ ]`（未完了）のリストの末尾に追加する
- 追加したタスクの前に、空行を1行入れて視覚的に区切る（既存タスクがある場合のみ）

### 4. 結果の報告

追加したタスクの一覧を表示し、plan.md のパスを示す。
