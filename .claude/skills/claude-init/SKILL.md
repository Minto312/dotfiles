# claude-init — ワークスペース永続化規約の初期化

このスキルは、新しいワークスペースを **sysmex 案件と同じ運用規約** で立ち上げる。
中心となる規約は次の 3 点:

1. **会話で得た情報・決定事項は常にファイルへ書き出して永続化する**（揮発させない）。
2. **`notes/` 以下に記録ファイルを作り**、種類ごとに整理して蓄積する。
3. **`CLAUDE.md` を適宜更新し**、参照しやすいよう常に最新の見取り図に保つ。現況（変わる状態）は CLAUDE.md の「### 現況」節に集約し、別ファイル（STATUS.md 等）には分散させない。

このスキル自体は「規約を敷く（雛形と索引を置く）」までを行う。以後の会話では、ここで作った CLAUDE.md の指示に従って Claude が自動的に notes/ へ書き出していく。

## 実行手順

### 0. 前提確認（破壊防止）

1. カレントディレクトリ（または引数で指定されたパス）に **既存の `CLAUDE.md` があるか** を確認する。
   - 既にあり、かつ `--force` が無い場合は **上書きしない**。内容を Read して要約し、「既に初期化済みに見える。追記・補強するか、雛形で作り直すか」をユーザーに確認する。
2. `git` リポジトリかどうかを確認する（`git rev-parse --is-inside-work-tree`）。リポジトリなら、グローバル規約に従い最初に **`git pull`** を提案/実行する。
3. `notes/` の有無を確認する。

### 1. ワークスペースの素性をヒアリング

引数に説明があればそれを起点にする。不足していれば、雛形を埋めるために最低限を確認する（推測で埋めず、分からない箇所は `❓未確認` として残す）:

- このワークスペースは何を扱うか（案件名・目的）
- 関係者・体制・責任分界（あれば）
- 確定している前提・制約
- 親ワークスペースや関連リポジトリの有無

回答が薄い場合でも初期化は進める。空欄は雛形に `❓` プレースホルダとして残し、後続会話で埋める前提にする。

### 2. ディレクトリの作成

```bash
mkdir -p notes
```

notes/ の索引は **CLAUDE.md の「ファイル一覧」に一本化**する（別途 `INDEX.md` は作らない）。ファイルが増えても CLAUDE.md を見れば見通せる状態に保つ。

### 3. CLAUDE.md の生成

下記テンプレートをベースに、ヒアリング内容で埋めて `CLAUDE.md` を生成する。
**「## 私からの指示」内の永続化ルールは必ず残す**（これがこの規約の核）。

```markdown
## 私からの指示

このワークスペースは，**<案件名・目的をここに>** のワークスペースです．
<1〜3行で何を扱うかの説明>

私との会話で得た情報や決定した事項などは常にファイルに書き出し永続化するようにしてください．
notes/ 以下にファイルを作成し，色々記録しておいてください．
CLAUDE.md は適宜更新し，参照しやすいように整えてください．
現況（変わる状態）は下記「### 現況」節に集約し、状態が変わったら追記でなく上書きで最新化してください．

## あなたのサマライズ

### このワークスペースの位置づけ

- <親ワークスペース・関連リポジトリとの関係／なければ「単独」>

### 前提（確定事項）

不変の前提・方針のみ。稼働状況など「変わるもの」は「### 現況」へ。

- <確定している前提を箇条書き。未確定は ❓ を付す>

### 現況

> 変わる状態のスナップショット。変わったらこの節を上書きで最新化する（古い状態は残さない）。

最終更新: <YYYY-MM-DD>

- <現在の稼働状況・進行中の作業・直近の予定など。初期化時点で無ければ「初期化のみ完了」>

### 責任分界 / 体制

- <関係者・責任分界。なければ省略可>

### ファイル一覧（notes/）

notes/ の索引は本節に一本化（別途 INDEX.md は作らない）。ファイルを追加・改名したらここを更新する。

- <作成した記録ファイルを追記していく>
```

### 4. 完了報告

- 作成/更新したファイル（`CLAUDE.md`, `notes/`）を列挙する。
- CLAUDE.md 内に残した `❓未確認` 項目があれば一覧化し、「次の会話で埋めていく」旨を伝える。
- 以後この規約に従って自動的に notes/ へ記録していくことを一言添える。

## 運用上のルール（このスキルが敷く規約の中身）

初期化後、Claude は次を継続的に守る:

- **記録の粒度**: 議事録・要件・決定事項・調査結果は、その都度 `notes/` のファイルに書き出す。会話だけで完結させない。
- **種類で分ける**: 議事録は議事録ファイルに「最新を上に蓄積」、要件は要件メモ、成果物は成果物として分離する。
- **状態マーカー**: 確定 `✅` / 要確認 `🔸` / 不明 `❓` を使い、未確定事項を明示する。相対日付は絶対日付に直して記録する。
- **索引の更新**: notes/ にファイルを追加・改名したら CLAUDE.md の「ファイル一覧」を更新する（索引は CLAUDE.md に一本化、別途 INDEX.md は作らない）。
- **現況の更新**: 稼働状況・進行中の作業・直近の予定が変わったら CLAUDE.md の「### 現況」節を上書きで最新化する（追記しない・古い状態を残さない）。現況の唯一の置き場は CLAUDE.md（STATUS.md 等は作らない）。
- **CLAUDE.md の鮮度**: 前提・スコープ・体制が変わったら CLAUDE.md を更新し、古い記述を残さない。

## 注意

- 既存ファイルを **勝手に上書きしない**。`--force` 指定時のみ雛形で再生成し、その際も既存内容を要約提示してから行う。
- ユーザーのグローバル規約（コミットメッセージに Claude 署名を入れない 等）に反しない。
- 機密ワークスペースでは notes/ の内容が外部同期・公開対象に含まれないか留意する。
