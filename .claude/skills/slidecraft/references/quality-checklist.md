# Slidecraft 品質チェックリスト

これは原則（principles.md）の *意図* を確認するためのレンズ。○×を機械的に付ける道具ではなく、「各スライドがその意図を満たしているか」を見るために使う。形式的に埋めて満足しない。

各スライドを以下5観点で確認する。S=そのまま / A=軽微修正 / B=要追記 / C=再構成。

## 採点5観点

| 観点 | 問い | NGの直し方 |
|---|---|---|
| **1スライド1メッセージ** | このスライドを1文で言える？ | 分割 / 片方を Appendix へ |
| **タイトル2層** | タイトル＝テーマ（名詞）／メッセージライン＝結論（動詞）になっている？ 重複していない？ | メッセージラインを結論文に書き直す。目次型をやめる |
| **So-What** | 事実の先に「だから何」がある？ | 結論ボックス（`▶`）を追加。「示唆」の語は使わない |
| **特徴→便益** | 「できる」だけで止まっていない？便益（できれば定量）がある？ | 「だから△△になる」を追記。仮値は「(仮)」で |
| **視認性** | 詰め込みすぎ／文字過多／表が長すぎ（>8行）ない？余白ある？ | 削る・分割・Appendix退避 |

## 出力フォーマット

```
## 品質レビュー
| # | タイトル | 1メッセージ | タイトル2層 | So-What | 特徴→便益 | 視認性 | 評価 | 直し |
|---|---------|:--:|:--:|:--:|:--:|:--:|:--:|------|
```

## 全体チェック（デッキ単位）

- [ ] 冒頭の目的（本日のゴール）と末尾の NA が呼応している
- [ ] 目的→結論→根拠→次の一手 の論理が飛んでいない
- [ ] 「なぜ我々／この案か」がどこかにある
- [ ] 確度ラベル（仮説 / 案 / 確定）が実態と合っている
- [ ] 相手が消極的な領域を過剰提案していない（温度感）
- [ ] 本編に技術詳細を入れすぎていない（Appendix活用）
- [ ] 機密数値（コスト・ROI）に機密ラベル
- [ ] KPI・効果が相手の物差し／上位目標に接続している
- [ ] 色はブランドに沿い、強調が意味のある所だけに使われている

## ラベルの使い分け

| ラベル | 使う場面 |
|---|---|
| （仮説） | ヒアリング前の仮説 |
| （案） | ヒアリング後の提案・すり合わせ |
| ラベルなし | 確定事項・実績・定量データ |
| 機密 | コスト・ROI 等、特に機密性の高いスライド |
| WIP | 作成中・確認中 |
